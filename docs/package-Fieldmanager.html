Exceptions summary
FM_Developer_Exception Exception class for this plugin's developer errors; mostly to differentiate in unit tests. These exceptions are meant to help developers write correct Fieldmanager implementations.
FM_Exception Exception class for this plugin's fatal errors; mostly to differentiate in unit tests.
FM_Validation_Exception Exception class for this plugin's validation errors; mostly to differentiate in unit tests. Validation errors in WordPress are not really recoverable.

_fieldmanager_registry A simple static registry for Fieldmanager to keep globals out.

fieldmanager_get_baseurl Get the base URL for this plugin.
fieldmanager_set_baseurl Tell fieldmanager that it lives somewhere other than wp-content/plugins
fm_add_script Wrapper to enqueue_scripts which uses a closure
fm_add_style Wrapper to enqueue_style which uses a closure
fm_get_post_meta Wrapper for get_post_meta which handles JSON quietly.
fm_is_post_edit_screen Cheap way to tell if we're looking at a post edit page. It's a good idea to use this at the beginning of implementations which use metaboxes to avoid initializing your Fieldmanager chain on every page load.
